1. Form HTML Dasar

Apa itu Form HTML?

Container untuk mengumpulkan dan mengirim data pengguna

Fungsi Utama:

Mengumpulkan Data

Nama, email, password, dll

Mengirim Data

Ke server untuk diproses

Validasi Input

Memastikan data sesuai format

Struktur Dasar:
<form
action="/submit"
method="POST"
>
<!-- Input elements here -->
</form>
Demo Interaktif - Form Sederhana:
Kode HTML:
<form action="/demo" method="POST">
<label>Nama:</label>
<input type="text" name="nama">
<button type="submit">Kirim</button>
</form>
Hasil Visual:

Atribut Penting pada Tag <form>

action

URL Tujuan

Menentukan ke mana data form akan dikirim

action="/submit-form"
method

Cara Pengiriman

GET (terlihat di URL) atau POST (tersembunyi)

method="POST"
enctype

Encoding Type

Format data saat dikirim (penting untuk upload file)

enctype="multipart/form-data"
target

Target Window

Di mana hasil form akan ditampilkan

target="_blank"

2. Metode GET vs POST

GET Method

Untuk pencarian dan mengambil data

Karakteristik:

Data Terlihat

Parameter tampil di URL address bar

Bisa Di-bookmark

URL dengan parameter bisa disimpan

Batasan Panjang

Maksimal ~2048 karakter di URL

Kapan Digunakan:
Form pencarian
Filter produk
Sorting data
Pagination
Contoh Interaktif - Form Pencarian:

URL yang akan terbentuk:

POST Method

Untuk mengirim data sensitif dan aman

Karakteristik:

Data Tersembunyi

Data dikirim melalui request body

Tanpa Batasan

Bisa mengirim data dalam jumlah besar

Lebih Aman

Cocok untuk data sensitif

Kapan Digunakan:
Form login
Form registrasi
Upload file
Pembayaran
Contoh Interaktif - Form Login:

URL yang terlihat:

https://website.com/login

Data email dan password dikirim secara tersembunyi melalui request body, tidak terlihat di URL

Perbandingan GET vs POST

Aspek GET POST
Lokasi Data URL Request Body
Keamanan Rendah Tinggi
Batasan Data ~2KB Tidak Ada
Bookmark
Cache Browser

3. Jenis-jenis Input

Ragam Input HTML

Berbagai jenis input untuk kebutuhan yang berbeda

Kategori Input:

Text Input

Untuk teks biasa seperti nama

Email & Password

Input khusus dengan validasi

Number & Date

Input numerik dan tanggal

Keunggulan:
Validasi otomatis
Keyboard mobile sesuai
User experience lebih baik
Keamanan data terjaga

Text Input

Input teks biasa untuk nama, alamat, dll.

<input
type="text"
name="nama"
placeholder="Masukkan nama"
required
>
Atribut Penting:
type="text" Jenis input teks
placeholder Teks bantuan
required Wajib diisi

Email Input

Input khusus email dengan validasi otomatis

<input
type="email"
name="email"
placeholder="nama@email.com"
required
>
Fitur Khusus:
@ Validasi format email
📱 Keyboard email di mobile
multiple Bisa beberapa email

Password Input

Input password yang menyembunyikan karakter

<input
type="password"
name="password"
minlength="8"
required
>
Keamanan:
•••• Karakter tersembunyi
minlength Panjang minimum
🔒 Tidak tersimpan di history

Number Input

Input khusus angka dengan kontrol naik/turun

<input
type="number"
name="umur"
min="1" max="100"
step="1"
>
Kontrol:
min/max Batas nilai
step Langkah kenaikan
↕️ Tombol naik/turun

Date Input

Input tanggal dengan date picker otomatis

<input
type="date"
name="tanggal_lahir"
min="1900-01-01"
max="2024-12-31"
>
Fitur:
📅 Date picker otomatis
YYYY-MM-DD Format standar
min/max Batasi rentang tanggal

Radio Button

Pilihan tunggal dari beberapa opsi

<input
type="radio"
name="gender"
value="L"
checked
> Laki-laki
Aturan:
name Sama untuk satu grup
value Nilai unik setiap opsi
⚪ Hanya satu terpilih

4. Select Option & Textarea

Elemen Input Lanjutan

Dropdown dan area teks untuk input yang lebih kompleks

Select Dropdown:

Pilihan Banyak

Hemat ruang untuk banyak opsi

User Friendly

Interface yang familiar

Multiple Selection

Bisa pilih beberapa sekaligus

Textarea:
Teks multi-baris
Ukuran dapat diatur
Ideal untuk komentar
Kontrol panjang teks

Select Option

Dropdown list untuk memilih dari banyak pilihan

<select name="kota" required>
<option value="">Pilih kota</option>
<option value="jkt" selected>Jakarta</option>
<option value="sby">Surabaya</option>
</select>
Atribut Select:
name Nama field untuk data
multiple Pilih beberapa opsi
size="3" Tampilkan 3 opsi
required Wajib memilih
Atribut Option:
value="jkt" Nilai yang dikirim
selected Terpilih default
disabled Opsi nonaktif
Demo Multiple Select:

Tahan Ctrl (Windows) atau Cmd (Mac) untuk pilih beberapa

Textarea

Area teks multi-baris untuk input panjang

<textarea
name="pesan"
rows="4" cols="50"
placeholder="Tulis pesan..."
required
>
Teks default di sini...
</textarea>
Atribut Penting:
name Nama field untuk data
rows="4" Tinggi dalam baris
cols="50" Lebar dalam karakter
placeholder Teks bantuan
required Wajib diisi
maxlength Batas karakter
Demo dengan Counter:
0/200
Resize dinonaktifkan dengan CSS resize: none Drag pojok kanan bawah untuk resize

Kapan Menggunakan Select vs Textarea

Kriteria Select Textarea
Jenis Input Pilihan Terbatas Teks Bebas
Ruang yang Dibutuhkan Hemat Butuh Ruang
Validasi Data Mudah Kompleks
Contoh Penggunaan Kota, Negara, Kategori Komentar, Pesan, Deskripsi

Tips & Tricks

Pro Tips:

  • • Gunakan placeholder untuk memberikan contoh input
  • • Tambahkan required untuk validasi otomatis
  • • Gunakan label yang jelas dan deskriptif
  • • Test form Anda dengan data yang berbeda

Hindari Kesalahan:

  • • Jangan lupa menutup tag dengan benar
  • • Pastikan name attribute unik untuk setiap input
  • • Radio button satu grup harus punya name sama
  • • Selalu test form sebelum submit
